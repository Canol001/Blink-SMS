Features

Everything you need
to run SMS at scale

Built for developers, operators, and businesses who need reliable, affordable SMS without the complexity.

REST API

Full-featured REST API with JSON payloads. Send single or bulk messages, schedule delivery, and manage queues — all from one endpoint.

Multi-SIM Support

Run multiple SIM cards from a single device. Blink auto-routes messages to balance load and maximise throughput.

Real-Time Delivery Reports

Get live status updates for every message — sent, delivered, failed. Webhook callbacks keep your system in sync automatically.

Inbound SMS Handling

Receive replies directly to your SIM number. Blink forwards incoming messages to your webhook instantly for two-way conversations.

Message Scheduling

Schedule messages minutes, hours, or days ahead. Perfect for reminders, campaigns, and time-sensitive alerts.

End-to-End Encryption

All API traffic is encrypted via TLS. Your API keys and device credentials are hashed and never stored in plain text.

For developers

One API call.
Delivered.

Our REST API is simple by design. Send a POST request using PHP and your message is on its way instantly.

Simple JSON API with predictable responses
Support for multiple gateway devices
Webhook support for delivery reports
Full API reference at docs.blinksms.io
Easy integration with PHP, Node.js, Python & more
send-sms.php
<?php

$api_key = "API_KEY";
$device_id = "DEVICE_ID";
$to = "+254712345678";
$message = "Hello from Blink SMS! Your OTP is 847921";

$data = [
"api_key" => $api_key,
"device_id" => $device_id,
"to" => $to,
"message" => $message
];

$ch = curl_init("https://sms.spacyfire.co.ke/api/v1/send.php");
curl_setopt($ch, CURLOPT_POST, true);
curl_setopt($ch, CURLOPT_POSTFIELDS, json_encode($data));
curl_setopt($ch, CURLOPT_HTTPHEADER, ['Content-Type: application/json']);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);

$response = curl_exec($ch);
curl_close($ch);

// Response from API
echo $response;

?>
